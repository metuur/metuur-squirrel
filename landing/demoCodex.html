The actual problem

The hardest bug is the handoff between you yesterday and you right now.

Squirrel is not another task app asking you to be better behaved. It is the system that remembers for you: no context lost between sessions, no decision forgotten, no deadline silently slipped, and no thought stranded on the wrong machine.

Re-entry is expensive

One interruption and the stack collapses. Reconstructing the why, the what, and the next action burns the best part of your attention.

Capture is fragmented

Ideas land in terminal buffers, notes apps, post-its, and half-written chats. The problem is not storing thoughts. It is finding them at the right moment.

Overwhelm blocks the start

When a task is too large to see clearly, the brain treats it like danger. The work sits there, fully visible, and still refuses to begin.

Deadlines stay silent too long

“Friday” feels abstract until the collision happens. Squirrel makes urgency visible before it becomes a crisis story.

What it actually is

One vault. Five surfaces. Zero cloud theater.

The product story is stronger than “task manager.” Squirrel is a system of coordinated surfaces around one local source of truth, so your context survives whether you are in the menu bar, terminal, browser, or an agent session.

The workflow

A lightweight loop built for attention reality, not discipline theater.

The system is small on purpose. Capture fast. Shrink the scary thing. Re-enter without ceremony. Let the product watch the edges while you work.

01 capture

Dump the thought before it evaporates.

Quick tasks, one-line notes, project-routed captures, or a freeform thought from the popup or agent.

› /sq-capture "email landlord about lease"

02 chunk

Turn resistance into a first move.

Large work gets sliced into small concrete chunks with a visible start point instead of a vague guilt cloud.

› /sq-chunk "quarterly report"

03 focus

Use AM / PM focus and quick-task guardrails.

Five active quick tasks max. Focus picks stay visible. Overdue and urgent work rises without you manually auditing every project.

04 recover

Re-enter exactly where you left off.

Squirrel restores the session trail, recent edits, next step, and surrounding context so interruption recovery takes seconds.

Feature depth from the specs

Not a mood board. A real product system with useful constraints.

Quick task stack

FIFO queue, hard cap of five active tasks, deterministic snooze return, and explicit guidance when the stack is full.

AM / PM focus

Focus picks, relative time updates, and menu-bar visibility keep the current commitment in front of you without opening a full workspace.

Mind journal

A recurring externalization prompt: what are you thinking, what are you doing, what mood are you in. The product treats transient mental state as first-class data.

Deadline surfacing

Pressing items and overdue work stay visible with relative urgency, instead of hiding in a quiet list until the damage is already done.

Runtime trust model

Local backend reachability, popup cache posture, tray state changes, and explicit offline semantics keep the system legible when something is unavailable.

Air-gap transfer

Context can move between separated environments through deliberate, hash-verified packages. The human stays the bridge by design.

Local-first proof

You can inspect it. Grep it. Back it up. Commit it.

Squirrel stores current state in plain Markdown, uses localhost for the web layer, and only leans on local SQLite where history genuinely matters. The product posture is visible in the file system, not hidden behind a pricing page.

  • Markdown is the source of truth. Projects, captures, chunks, and decisions remain human-readable files.
  • `127.0.0.1` is canonical. The popup and web UI only talk to the local backend.
  • No cloud account required. Privacy is the floor, not an advanced setting.
