# Bug Hunt — 2026-06-10

Scope: `apps/desktop` (+ `packages/design-system`), `apps/cli` + `apps/backend`, `scripts/` + `installer/`, `agent-pack/`. Four parallel review agents; the highest-impact claims were verified directly against the source (marked **verified**). `docs/` is prose only and was used as reference for intended behavior.

---

## Confirmed bugs (verified in source)

### 1. `build-manual-zip.sh` crashes immediately — `say` never defined — HIGH ✅ verified
- **File:** `scripts/build-manual-zip.sh:87` (also lines 329–338)
- The helper block (lines ~62–68) defines `ok/info/warn/die/hdr/run` but not `say`. With `set -euo pipefail` (line 34), the first `say ""` at line 87 is "command not found" and the script dies before Step 1. The manual-install zip cannot currently be built.
- **Fix:** add `say() { printf '%s\n' "$*"; }` next to the other helpers.

### 2. Copilot prompt install raises `TypeError` — HIGH ✅ verified
- **File:** `apps/cli/squirrel:418` (signature at line 323)
- `_copy_prompt_for_copilot` calls `_inject_copilot_frontmatter(body, meta, kind="prompt")` but the function signature is `(body, source_meta)` — no `kind` param. Every non-dry-run copy of a command prompt file during `squirrel install --agent copilot` crashes. The skills path (line 386) calls it without `kind` and is fine.
- **Fix:** drop the `kind="prompt"` argument, or add `kind: str = "agent"` to the signature.

### 3. Tray reminders fetch hardcodes port 3939 — HIGH ✅ verified
- **File:** `apps/desktop/src-tauri/src/tray_alerts.rs:527`
- `fetch_reminders` uses literal `http://127.0.0.1:3939/api/reminders` while every sibling fetch uses `BACKEND_ORIGIN` (overridden to 3940 in dev builds). In dev, reminders silently come from the wrong backend (or fail) while the rest of the tray works.
- **Fix:** `format!("{BACKEND_ORIGIN}/api/reminders")`.

### 4. Reminder daemon's scanner error handler is dead code — HIGH ✅ verified
- **File:** `agent-pack/companions/macos-reminders/reminder-daemon.sh:418-421`
- `SCAN_OUTPUT=$(python3 "$SCANNER" ...)` followed by `if [ $? -ne 0 ]` under `set -euo pipefail` (line 16): a failing scanner exits the shell before the `if` runs. The daemon dies silently with no log entry.
- **Fix:** `SCAN_OUTPUT=$(...) || { log "deadline_scanner error: $SCAN_OUTPUT"; exit 0; }`

## Desktop app (`apps/desktop`)

### 5. `useQuickTasks` — stale `reload` closure captured by event listener — HIGH
- **File:** `src/hooks/useQuickTasks.ts:35-46, 59-73`
- The `squirrel:notif-updated` listener is registered with `[]` deps and captures the initial `reload` whose closure has `data === null`. Daemon events can reset the widget to `EMPTY` on a failed fetch instead of keeping last-known data. The `eslint-disable` at line 72 suppressed the symptom, not the cause.
- **Fix:** use `setData(prev => prev ?? EMPTY)` in the fallback and remove `data` from `reload`'s dependency array.

### 6. `HandshakeBanner` Close button dismisses a terminal safety block — MEDIUM
- **File:** `src/components/HandshakeBanner.tsx:158-168`
- The file's own header comment says the refusal is terminal for the process lifetime ("no dismiss; only Quit or relaunch"), yet the Close button calls `setCause(null)`, exposing a non-functional popup whose API calls all fail.
- **Fix:** remove the button or replace with a Quit action.

### 7. Health-loop respawn never surfaces `RefusedAdoption` — MEDIUM
- **File:** `src-tauri/src/backend_supervisor.rs:629-635`
- When the post-failure respawn ends in `RefusedAdoption`, `notify_refusal` is not called (unlike the `wait_for_ready` path), so the user sees an error tray icon with no HandshakeBanner explaining why.
- **Fix:** call `notify_refusal(app)` when the respawned mode is `RefusedAdoption`.

### 8. `markAllRead` zeroes state blindly + unhandled rejection — MEDIUM
- **Files:** `src/hooks/useNotifications.ts:57-61`, `src/components/NotificationCenter.tsx:121`
- After POST `/notifications/read-all`, local state is set to `{items: [], unreadCount: 0}` without refetching; the `void markAllRead()` call site has no catch, so a failed POST is an unhandled rejection with stale UI.
- **Fix:** try/catch at the call site; refetch after success instead of zeroing.

### 9. `dialogs_today` daily cap can be exceeded across phases — LOW-MEDIUM
- **File:** `src-tauri/src/tray_alerts.rs:560-613`
- The cap is read in two separate lock acquisitions (`select_candidates` and `select_reminder_candidates`) within one `check_notifications` cycle, so a cycle can exceed `MAX_DIALOGS_PER_DAY`.
- **Fix:** read the count once at cycle start and thread it through both phases.

## Python CLI / backend (`apps/cli`, `apps/backend`)

### 10. `config_loader._atomic_write` — shared `.tmp` path races — MEDIUM
- **File:** `apps/cli/lib/config_loader.py:522` (same pattern in `apps/cli/lib/reminder_writer.py:161-180`)
- Temp path is always `config.toml.tmp`; concurrent writers race on the same file and `os.replace`, which can corrupt `config.toml`. `intent_parser._atomic_write_bytes` already does this correctly with `tempfile.mkstemp`.
- **Fix:** use `tempfile.mkstemp(dir=path.parent)` in both places.

### 11. Quick-task snooze TOCTOU — double `_snooze_count` read — MEDIUM
- **File:** `apps/cli/lib/quick_task_writer.py:216-223`
- `_snooze_count(path)` is called for the guard and again for the write; rapid double-snoozes both pass the `>= MAX_SNOOZES` check.
- **Fix:** read once into a local, use for both guard and write.

### 12. Deadline urgency uses naive datetimes — DST skew — MEDIUM
- **File:** `apps/cli/lib/deadline_scanner.py:54-58` (and `datetime.now()` at ~77)
- Naive-datetime subtraction is off by one hour across DST transitions, which can flip `critical` ↔ `urgent` near the boundary.
- **Fix:** use aware datetimes (`datetime.now().astimezone()` consistently).

### 13. `?limit=-1` means unlimited; limit interpolated into SQL — MEDIUM
- **File:** `apps/backend/server.py:1910`
- `limit` is int-parsed but f-string-interpolated (`LIMIT {limit}`); SQLite treats negative LIMIT as no limit.
- **Fix:** validate `limit > 0` and bind as a parameter.

### 14. Path-traversal gate checked before URL decoding — MEDIUM (defense-in-depth)
- **File:** `apps/backend/server.py:270-277`
- The `..` segment check runs on the raw URI, so `/%2e%2e/` passes. No current handler uses the path for filesystem ops, but the gate doesn't enforce what it claims.
- **Fix:** `urllib.parse.unquote` before the segment check.

### 15. launchd token generation umask edge — LOW
- **File:** `apps/backend/launchd/install.sh:88-92`
- `umask` is restored before `chmod 600`; with a 000 umask the token is briefly world-readable.
- **Fix:** keep `umask 077` in effect through the `chmod`.

## Build/release scripts and installer (`scripts/`, `installer/`)

### 16. Installer rollback can load a half-written plist — MEDIUM
- **File:** `installer/install.sh:152`
- The `ERR`-trap rollback runs `launchctl load "$PLIST_PATH"` even if the failure happened mid-plist-write, registering a corrupt LaunchAgent that persists across reboots.
- **Fix:** `plutil -lint "$PLIST_PATH"` before loading (or don't reload on rollback).

### 17. `build-manual-zip.sh` hard-depends on a prior `build-dmg.sh` run — MEDIUM
- **File:** `scripts/build-manual-zip.sh:274-275`
- `AGENT_PACK_SRC="$ROOT/dmg-staging/agent-pack"` dies if absent, but `dmg-staging/` only exists after `build-dmg.sh`. The config-example two lines later already falls back to `$ROOT/agent-pack` — the agent-pack source should too. (Currently masked by bug #1, which kills the script earlier.)
- **Fix:** fall back to `$ROOT/agent-pack` before dying.

### 18. `eval`-based `run()` pattern breaks on single quotes in paths — LOW-MEDIUM
- **Files:** `scripts/build-pkg.sh:126` (stapler), `scripts/publish-release.sh:175` (FLAGS concatenation), and the shared `run() { eval "$*"; }` in several scripts
- Any path containing `'` corrupts the eval'd command; in `build-pkg.sh` that would silently skip stapling the notarization ticket. The string-concatenated `$FLAGS` in `publish-release.sh` is the same fragility.
- **Fix:** call signing/stapling/release commands directly with quoted args (arrays for flags) instead of routing through `eval`.

### 19. npm token written to tmpdir with default permissions — LOW
- **File:** `scripts/publish-npm.sh:128`
- `.npmrc` containing a live `gh auth token` survives a SIGKILL in `/tmp` with default perms.
- **Fix:** `install -m 600 /dev/null "$WORK_DIR/.npmrc"` before writing.

## Agent-pack (`agent-pack/`)

### 20. Scanner JSON passed via environment variable — silent `E2BIG` on large vaults — HIGH
- **File:** `agent-pack/companions/macos-reminders/reminder-daemon.sh:435-471`
- `RECORDS=$(SCAN_OUTPUT="$SCAN_OUTPUT" python3 ...)` passes the full scanner JSON as an env var; macOS caps env+args (~256 KB). On large vaults the exec fails, `2>/dev/null` hides it, `RECORDS` is empty, and zero banners are emitted with nothing in the log.
- **Fix:** write `SCAN_OUTPUT` to a temp file and pass the path as `argv`.

### 21. `cadence_minutes` config is ignored by launchd — MEDIUM
- **Files:** `agent-pack/companions/macos-reminders/plist.template:16`, `companions/macos-reminders/install.sh:63-67`
- `StartInterval` is hardcoded to 7200s; the installer never substitutes a cadence token, so `cadence_minutes < 120` in `config.toml` has no effect on actual wake frequency.
- **Fix:** add a `__CADENCE_SECONDS__` template token and substitute `cadence_minutes * 60` at install time.

### 22. Copilot stdin hook truncates long prompts mid-JSON — MEDIUM
- **File:** `agent-pack/companions/copilot/hook-adapter-stdin.sh:43-60`
- The 4096-byte `dd` cap truncates long prompts; `json.loads` then fails silently and `USER_PROMPT` is empty, so prompt-aware hooks never trigger on long inputs. Payload is also passed as `argv[1]` rather than stdin.
- **Fix:** raise/remove the read cap and pipe the payload to Python via stdin.

### 23. `read_config` key interpolated unescaped into `grep -E` — LOW
- **File:** `agent-pack/companions/macos-reminders/reminder-daemon.sh:197-203`
- All current keys are literals, but `.` in a key would match any character.
- **Fix:** use `grep -F` or escape the key.

### 24. Literal `\n` shown in fallback dialog — LOW (cosmetic)
- **File:** `agent-pack/companions/macos-reminders/reminder-daemon.sh:126-128`
- `"...\n\n"` in plain double quotes is not interpreted; the dialog displays literal `\n\n`.
- **Fix:** `local prefix=$'...\n\n'`.

### 25. `install_canonical` always writes to `~/.claude/` regardless of agent selection — JUDGMENT CALL
- **File:** `agent-pack/install.sh:411-421`
- Runs unconditionally (lib discovery), even for Standalone/Copilot-only selections; summary screen shows the plugin path unconditionally. May be by design, but it ignores user consent and `WANT_CLAUDE` is never consulted.
